### Overview

* The `Serverless Application Model (sam)` is a toolkit designed to improve the developer experience of building serverless applications on aws
* When working with `sam` you must first define a `sam template`
    - this is usually a yaml file that has all the configurations that defines all the serverless application resources:
        - lambda functions
        - api gateways
        - eventbridge
        - etc.

### SAM cli

* The `sam cli` is a tool that is used to build, test, debug and deploy serverless applications on aws
    - `sam init`: creates a new serverless project with your pre-configured template for your preferred runtime
    - `sam build`: builds the serverless applications deployment package
    - `sam local invoke`: invokes a local `lambda function` once
    - `sam package`: packages a serverless application for deployment
    - `sam deploy`: deploys a serverless application to aws
        * pulls package from s3 location and passes it to cloudformation for deployment
    - `sam logs`: fetches the logs for the `lambda function`
    - `sam validate`: validates your `sam template`
    - `sam publish`: publish `sam package` to serviceless application repository (s3 bucket)

### Serverless Application Repository

* The `serverless application repository(sar)` is a managed repository for serverless applications
    - similar to artifactory hub for helm, it's the same thing but for serverless applications
    - they can be shared as they can be public or private
