## AWS Batch
- [Overview](#overview)
- [Components](#components)

### Overview

* AWS `Batch` is a service that helps you with batch jobs
    - you simply provide aws with the job you want to run and aws takes care of scheduling the deployment of these jobs based on the pool of compute resources provided
    - it also allows you to queue jobs and set a priority level on batches that run
* The lifecycle of a batched job are as follows:
    - `submitted`: your job has been submitted to batch
    - `pending`: job is in the queue
    - `runnable`: job can be run, waiting for compute resources to be available
    - `starting`: job is starting
    - `running`: job is current running
    - `failed`: batch job has failed
    - `succeeded`: batch job has completed successfully

### Components

* Components of a batch are as follows:
    - `job definition`: template that describes how a job will be run
        * what docker image will be used
        * what resources will it need
        * command to run
        * env vars, etc
    - `job submission`: once job is defined it can be submitted to batch
        * each job is an instance of the `job definition`
    - `job queue`: this is where jobs will be submitted to
    - `job scheduler`: determines when and where to run jobs
        * based on priority of a job, it will schedule when they run
    - `job execution`: jobs are run in docker containers
        * for isolation
    - `compute environments`: collection of compute resources where jobs will run
        * support:
            - ec2 on demand
            - ec2 spot
            - fargate
        * `batch` will auto scale resources based on volume and requirements of jobs in the queue
