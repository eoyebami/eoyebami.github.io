### Overview

![alt text](images/mq/image.png)

* AWS `Amazon MQ` is a managed message broker service for `Apache Active MQ` and `RabbitMQ`
    - it simplifies the setup, provisioning, and maintenance by handling hardware provisioning, software patching, and failure recovery
* A message broker is a type of software that enables different applications to communicate asynchronously
* Its very similar to `sqs`, however its for applications that are using an existing `mq` service and are migrating to the cloud
    - simplifies the process of using a managed mq rather than rewriting code to uss `sqs` and `sns`

### Features

![alt text](images/mq/image-1.png)
![alt text](images/mq/image-2.png)
* `Engine Support`: choose between `activemq` and `rabbitmq`
* `Deployment Modes`: supports single instance brokers and active/standby brokers for HA across multiple `AZs` 
* `Message Queuing Decoupling`: similar to `sqs`
