## Subnets
- [Overview](#overview)
- [Configuration](#configuration)

### Overview

* `Subnets` are a group of ip addresses in your `vpc`
* They also reside in a single `AZ`, so by specifying a `subnet` where a device will be deployed, you are also telling aws what `AZ` that device will be deployed into
* `Subnets` can be made public or private to allow external access to resources within them
    - `Public Subnet`: you need the internet to be able to access devices
        * `Route Table` wih `igw` will be attached to this subnet
    - `Privae Subnet`: internet should not be able to access these devices (databases)
        * `Route Table` with `nat` will be attached to this subnet to allow outbound access

### Configuration

* `Subnets` within a `vpc` must be with the `cidr block` range of the `vpc`
    - Example:
        * Vpc Cidr Range: `192.168.0.0/16`
        * Subnet 1 Cidr Range: `192.168.10.0/24`
    - Block size must be between `/16` and `/28`
    - First 4 ip addr of a `subnet` are reserved and cannot be used
        * `192.168.10.0`: reserved as network address
        * `192.168.10.1`: reserved for aws vpc router
        * `192.168.10.2`: reserved for aws dns
        * `192.168.10.3`: reserver for aws future use
    - Last ip addr of a `subnet` is reserved as the broadcast address
        * `192.168.10.255`
* `Subnets` cannot overlap with other `subnets` in a `vpc`, ranges must not collide
* `Subnets` can be configured with IPv6 and only IPv6
* `Subnets` can communicate with one another within a `vpc`
* `Subnets` can be configured to auto assign public IPv4/6 and private ip addr
