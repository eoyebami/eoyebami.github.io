## VPCS
- [Overview](#overview)

### Overview

* A `VPC` or a virtual private cloud, is a secure isolated network segmented hosted in AWS. It allows us to isolated resources from one another within an account.

* Through `vpcs` we get full control of networking within the cloud
  - define subnetting (ip address)
  - configure routing (route tables)
    * define where and what path packets take as they go through your account
  - define firewalls (NACLS and Security groups)
    * define access to network itself and specific resources
  - customize traffic in and out of `vpc` (gateways)

* `VPCs` are specific to a region, isolated to a region and cannot span multiple regions
* `VPCs` by defaulyt act as a network boundary, `vpcs` will not be able to communicate between one another. This needs to be explicitly enabled

### Cidr Blocks

* Every `vpc` has a range of ip addr assigned to it called a `cidr block`
  - Can be from ranges `/16` and `/28`
  - Cou can measure your ranges [here](https://cidr.xyz/)
* It is possible to enable an optional secondary IPv4 Block for your `vpc` as well as an IPv6/56 Block
    - You can hve up to 5 IPV6 blocks but this limit is adjustable

### Default and Custom VPCs

* `Default VPCs` are as the name suggests, they are created by default in each region, that you have access to, within an AWS account
  - They will have default configurations
    1. Will use a `172.31.0.0/16` cidr block range
    2. Each AZ will have a default `subnet` with a `/20` block range
    3. An `Internet Gateway(igw)` will be attached to the vpc
    4. A `route` that points all traffic (`0.0.0.0/0`) to that `igw` will be created
      - A devices in these default `subnets` will have access to the internet
      - All devices in the internet will have access to devices in your vpc (default `security group` will restrict this though)
    6. A default `security group` will be created
      - Will allow all outbound traffic
    7. A default `network access control list (NACL)` will be created
      - Will allow all inbound and outbound traffic

* `Custom VPCs` are created by the enduser, where you define any and all configurations yourself
